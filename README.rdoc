Rails 4 out-of-the-box is that you don't need to include the
ActiveUUID::UUID module in the models and when you create the migration,
instead of using id: false you would use id: :uuid:

class CreateStories < ActiveRecord::Migration
  def change
    create_table :stories, id: :uuid do |t|
      # To handle the relationship with the authors table
      t.uuid :author_id

      t.timestamps
    end
  end
end

Create roles use Polymorphic associations can be perplexing.
I to set it up in Active Record and then move to the controller and view layer.

Using gem package Rolify

  bash> rails g rolify Role User
  bash> rake db:migrate

To define a global role: admin for user with id:1

  pry> User.all
  pry> User.find(1)
  pry> user = User.find(1)
  pry> user.add_role :admin

To define a role scoped to a resource instance:

  pry> User.all
  pry> User.find(2)
  pry> user.add_role :moderator, Forum.first

To define a role scoped to a resource class:

  pry> User.all
  pry> User.find(3)
  pry> user = User.find(3)
  pry> user.add_role :moderator, Forum

That's it !
